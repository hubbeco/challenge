# Full Stack Challenge — Zedia

## Contexto (história)

Você entrou como dev em um time que está construindo um sistema interno de comunicação por e-mail.

A empresa possui vários usuários cadastrados e, todos os dias, diferentes áreas precisam enviar mensagens padronizadas como:

- Aviso de incidente (ex: instabilidade em VPN, e-mail, ERP)
- Confirmação de recebimento de equipamento (notebook, crachá, etc.)
- Aviso de manutenção (sistemas fora do ar em determinado horário)
- Comunicado de evento interno (happy hour, comemoração)
- Convite para reunião geral (all-hands, town hall)
- Outras

Para evitar que cada time copie e cole texto manualmente, foi decidido criar uma plataforma simples chamada **Mensageiro**, onde usuários podem:

- Criar templates de e-mail
- Preencher template com variáveis
- Enviar para outros usuários do sistema
- Garantir regras mínimas de segurança

Seu objetivo é entregar uma aplicação completa (**API + Frontend**) que cubra esse fluxo.

---

## Objetivo do desafio

Desenvolver uma aplicação completa (**API + Frontend**) onde usuários cadastrados podem:

- Criar conta e autenticar
- Criar e gerenciar templates de e-mail
- Preencher um template com variáveis
- Disparar o e-mail para outros usuários cadastrados no sistema

A aplicação deve possuir:

- Rotas públicas e privadas
- Documentação da API
- Frontend funcional
- Execução via Docker

---

## Escolhas técnicas (livres)

A escolha de stack, framework, banco de dados e arquitetura é livre.

O candidato pode utilizar, por exemplo:

- qualquer linguagem (Go, Java, Python, etc.)
- qualquer banco (PostgreSQL, MySQL, SQLite, MongoDB, etc.)
- qualquer arquitetura (MVC, Clean Architecture, Hexagonal, etc.)

O importante é que a solução entregue atenda os requisitos e regras do desafio, com organização e clareza.
